# EEG/ERP Analysis

## Purpose

This skill provides a complete methodological framework for EEG/ERP research, from initial experimental design through final statistical analysis. It consolidates three specialized sub-skills that cover the full research pipeline:

1. **Paradigm Design** — Designing EEG experiments optimized to isolate specific ERP components
2. **Preprocessing** — Transforming raw EEG recordings into clean, analysis-ready data
3. **Statistical Analysis** — Measuring and testing ERP components with appropriate methods

EEG/ERP research requires domain-specific expertise at every stage. A competent programmer without neuroscience training would not know that a 1 Hz high-pass filter improves ICA decomposition but distorts slow ERP components, that N2pc paradigms require 150+ trials while P3b paradigms need only 50, or that selecting time windows based on observed differences inflates Type I error. This skill encodes the methodological knowledge required to conduct rigorous EEG/ERP research.

## When to Use This Skill

Use this skill when:

- Designing a new EEG/ERP experiment from scratch
- Setting up a preprocessing pipeline for EEG data
- Analyzing ERP components and choosing statistical methods
- Reviewing or troubleshooting an existing EEG/ERP analysis
- Making methodological decisions about timing, trial counts, filtering, or artifact rejection
- Selecting appropriate measurement approaches for specific ERP components

## Research Planning Protocol

Before executing the domain-specific steps below, you MUST:

1. **State the research question** — What specific cognitive process or neural mechanism is this EEG/ERP study targeting?
2. **Justify the method choice** — Why EEG/ERP (not fMRI, MEG, behavior-only)? What alternatives were considered?
3. **Declare expected outcomes** — Which ERP component(s) do you expect to observe, with what polarity, latency, and scalp distribution?
4. **Note assumptions and limitations** — What does this method assume? Where could it mislead?
5. **Present the plan to the user and WAIT for confirmation** before proceeding.

## Integrated Workflow Overview

The complete EEG/ERP research pipeline follows this sequence:

```
1. DESIGN PHASE (references/eeg-paradigm-designer.md)
   ├─ Identify target ERP component
   ├─ Select paradigm type
   ├─ Configure timing parameters (ISI, jitter, epoch length)
   ├─ Calculate required trial counts
   ├─ Design difference waveforms and control conditions
   └─ Choose electrode montage density

2. DATA COLLECTION
   └─ Run participants (not covered by this skill)

3. PREPROCESSING PHASE (references/eeg-preprocessing-pipeline.md)
   ├─ Import and inspect raw data
   ├─ Remove bad channels
   ├─ High-pass filter (0.1 Hz for analysis; 1 Hz for ICA)
   ├─ Remove line noise (CleanLine/ZapLine)
   ├─ Re-reference (average reference for ≥64 ch)
   ├─ ICA decomposition and artifact removal
   ├─ Interpolate bad channels
   ├─ Epoch and baseline correct
   └─ Epoch rejection by amplitude threshold

4. ANALYSIS PHASE (references/erp-analysis.md)
   ├─ Identify ERP components in grand average
   ├─ Define time windows and electrode ROIs
   ├─ Choose amplitude measure (mean/peak/area)
   ├─ Select statistical framework (ANOVA vs. mass univariate)
   └─ Report results with effect sizes and preprocessing details
```

## Cross-Stage Dependencies and Critical Decision Points

### Design → Preprocessing

**Trial count planning**: The paradigm design determines how many trials you collect, but preprocessing artifact rejection will remove 20-30% of trials. Design for 25% attrition (Luck, 2014).

**Filter planning**: If you plan to study slow components (P3b, N400, P600, CNV), ensure your recording system supports DC or very low high-pass cutoffs (≤0.01 Hz), and plan to use 0.1 Hz high-pass in preprocessing (Tanner et al., 2015).

**Lateralized components**: If your paradigm targets lateralized components (N2pc, LRP, N170), use ≥64 channels and plan for average reference or CSD transformation (Luck, 2014).

### Preprocessing → Analysis

**Filter effects on components**: A 1 Hz high-pass filter distorts slow ERP components. If you filtered at 1 Hz during preprocessing, you cannot reliably measure P3b, N400, P600, or CNV (Tanner et al., 2015).

**Reference scheme effects**: Average reference requires ≥64 channels. If you used average reference with <64 channels, topographic maps may be biased (Dien, 1998).

**Trial count verification**: After artifact rejection, verify you have ≥30 trials per condition (Boudewyn et al., 2018). If not, you may need to adjust artifact rejection criteria or collect more data.

### Design → Analysis

**Component identification**: The same voltage deflection at the same latency can reflect different components depending on the paradigm. A frontal negativity at 200-300 ms could be N2, N2pc, or MMN depending on the task (Luck, 2014).

**Difference waveforms**: The control condition you designed determines what your difference waveform isolates. Ensure your statistical analysis tests the theoretically meaningful contrast (Luck, 2014; Kappenman & Luck, 2010).

## Common Cross-Stage Pitfalls

### Pitfall 1: Inadequate Trial Counts After Artifact Rejection

**Problem**: You designed for 50 trials per condition (adequate for P3b), but after artifact rejection you have only 20 trials (inadequate).

**Solution**: Design for 25-30% attrition. If you need 50 clean trials, collect 65-70 trials per condition (Luck, 2014).

**Relevant sections**:
- `references/eeg-paradigm-designer.md` → Step 3: Determine Trial Counts

### Pitfall 2: Filter Settings Incompatible with Target Component

**Problem**: You designed a paradigm to measure P600 (a slow component), but preprocessed with a 1 Hz high-pass filter, which distorts slow components.

**Solution**: Use 0.1 Hz high-pass for ERP analysis. Use 1 Hz only for ICA training, then apply ICA weights to 0.1 Hz-filtered data (Winkler et al., 2015; Tanner et al., 2015).

**Relevant sections**:
- `references/eeg-paradigm-designer.md` → Step 6: Verify Against Common Pitfalls (item 10)
- `references/eeg-preprocessing-pipeline.md` → Step 3: High-Pass Filtering

### Pitfall 3: Stimulus Confounds Masquerading as Cognitive ERPs

**Problem**: Your N170 difference (faces vs. houses) is driven by low-level visual differences (luminance, contrast), not face-specific processing.

**Solution**: Equate low-level stimulus properties across conditions or use control stimuli that isolate the cognitive dimension (Luck, 2014; Kappenman & Luck, 2010).

**Relevant sections**:
- `references/eeg-paradigm-designer.md` → Step 4: Design the Difference Waveform
- `references/eeg-paradigm-designer.md` → Step 6: Verify Against Common Pitfalls (item 2)

### Pitfall 4: Double-Dipping in Time Window Selection

**Problem**: You selected a time window by looking at where conditions differ, then tested whether conditions differ in that window (circular analysis).

**Solution**: Use a priori time windows from literature, or use a collapsed localizer (average across conditions), or use mass univariate methods (Luck & Gaspelin, 2017).

**Relevant sections**:
- `references/erp-analysis.md` → Time Window and ROI Selection
- `references/erp-analysis.md` → Common Pitfalls (item 1)

### Pitfall 5: Overlapping ERPs from Fast Presentation

**Problem**: Your ISI is 800 ms, but your target component (P3b) extends to 600 ms, causing ERP overlap that distorts measurements.

**Solution**: Use ISI ≥ 1200 ms for P3b, or use overlap correction methods (ADJAR, unfold toolbox) (Woldorff, 1993; Ehinger & Dimigen, 2019).

## Minimum Reporting Checklist (All Stages)

When reporting EEG/ERP research, include:

### Design Stage
- [ ] Target ERP component(s) and theoretical rationale
- [ ] Paradigm type and key manipulations
- [ ] Timing parameters (ISI, jitter, stimulus duration)
- [ ] Number of trials per condition (designed and collected)
- [ ] Electrode montage (number of channels, system)

### Preprocessing Stage
- [ ] Sampling rate (original and any downsampling)
- [ ] Filter settings (type, cutoffs, order, transition bandwidth)
- [ ] Line noise removal method
- [ ] Re-referencing scheme
- [ ] Bad channel identification criteria and number removed
- [ ] ICA algorithm and number of components removed (mean and range)
- [ ] Epoch window and baseline correction window
- [ ] Artifact rejection criteria and proportion rejected
- [ ] Minimum retained trials per condition (mean and range)

### Analysis Stage
- [ ] ERP component identification criteria
- [ ] Time window and electrode ROI (with justification)
- [ ] Amplitude measure (mean, peak, area, latency)
- [ ] Statistical test and correction method
- [ ] Effect sizes (partial η², Cohen's d)
