# Cognitive Science Statistics

## Purpose

This skill provides a unified entry point for statistical analysis, power planning, visualization, and specialized methods in cognitive science and neuroscience research. It routes users to the appropriate specialized methodology based on their research stage, data type, and analysis goals.

A competent programmer without domain training would apply generic statistical methods without understanding field-specific requirements: when to use mixed-effects models vs. ANOVA, how neuroimaging power differs from behavioral power, why signal detection theory matters for discrimination tasks, or which eye-tracking measures reflect which cognitive processes. This router skill directs users to domain-validated methods for each scenario.

## When to Use This Skill

Use this skill when you need guidance on:

- **Study planning**: Sample size determination, power analysis for grants or preregistration
- **Data analysis**: Choosing statistical models, handling reaction times, analyzing accuracy data
- **Specialized paradigms**: Signal detection tasks, eye-tracking reading studies, neuroimaging analyses
- **Visualization**: Creating publication-quality figures following field conventions
- **Reporting**: Effect sizes, statistical reporting standards, figure formatting

## Research Planning Protocol

Before executing any statistical analysis, you MUST:

1. **State the research question** — What specific hypothesis is being tested?
2. **Justify the method choice** — Why this statistical approach? What alternatives were considered?
3. **Declare expected outcomes** — What pattern of results would support vs. refute the hypothesis?
4. **Note assumptions and limitations** — What does this method assume? Where could it mislead?
5. **Present the plan to the user and WAIT for confirmation** before proceeding.

## Method Overviews

### Statistical Modeling (references/cogsci-statistics.md)

**When to use**: Analyzing behavioral data with repeated measures, crossed random effects (subjects and items), or non-normal distributions.

**Key topics**:
- Repeated-measures ANOVA vs. mixed-effects models (when items are random)
- Random effects structure (maximal models, convergence issues)
- Handling reaction time distributions (Gamma GLMM, transformations)
- Multiple comparison correction (FDR, cluster-based, Bonferroni-Holm)
- Bayesian alternatives (Bayes factors, when to use)
- Effect size reporting (Cohen's d, partial eta-squared, R-squared for mixed models)

**Critical insight**: If stimuli are sampled from a population (words, faces, scenes), you must use mixed models with crossed random effects for subjects AND items (Clark, 1973; Baayen et al., 2008).

### Power Analysis (references/cogsci-power-analysis.md)

**When to use**: Planning sample size for behavioral or EEG studies; need paradigm-specific effect size priors.

**Key topics**:
- Effect size priors by modality (behavioral d = 0.40 median; ERP d = 0.50-1.00)
- Sample size recommendations (behavioral: 30-50 per group for d = 0.5; ERP: 25-40 subjects)
- Simulation-based power for complex designs (simr, Superpower)
- Within-subjects vs. between-subjects power differences
- Trial count requirements for EEG/ERP (minimum 30 trials per condition)

**Critical insight**: Do not use Cohen's generic benchmarks (0.2/0.5/0.8). Use paradigm-specific meta-analytic estimates (Brysbaert, 2019).

### Neuroimaging Power (references/neuroimaging-power-guide.md)

**When to use**: Quick benchmarks for fMRI/EEG sample size; understanding why neuroimaging power differs from behavioral power.

**Key topics**:
- Why standard power analysis fails for neuroimaging (multiple comparisons, spatial correlation)
- Typical fMRI effect sizes (voxel-level d = 0.5-1.0; brain-behavior r = 0.1-0.3)
- Sample size benchmarks (within-subject activation: 25-30; between-group: 40-50 per group)
- Multiple comparison correction impact on power (FDR vs. FWE vs. cluster-based)
- Test-retest reliability and individual differences (ICC = 0.3-0.6 for task fMRI)

**Critical insight**: Brain-behavior correlations require N > 2000 for replicable whole-brain results (Marek et al., 2022).

### Neuroimaging Sample Size Calculator (references/neuroimaging-sample-size-calculator.md)

**When to use**: Simulation-based power analysis for fMRI using pilot data or published effect size maps.

**Key topics**:
- Pilot-data-based simulation workflow (fMRIpower, NeuroPowerTools)
- Effect size extraction from published results (converting t-values to Cohen's d)
- ROI-based power shortcuts (when whole-brain simulation is impractical)
- Registered report requirements (pre-specification, sensitivity analysis)
- Attrition planning (10-20% exclusion rate for fMRI)

**Critical insight**: Pilot effect sizes from N < 20 are inflated; deflate by 25-50% (Button et al., 2013).

### Signal Detection Theory (references/signal-detection-analysis.md)

**When to use**: Analyzing detection, discrimination, or recognition memory tasks where sensitivity and bias must be separated.

**Key topics**:
- d-prime calculation (z(Hit Rate) - z(False Alarm Rate))
- Bias measures (criterion c, likelihood ratio beta, relative criterion c')
- Paradigm-specific formulas (yes/no, 2AFC, rating scale, same-different)
- Extreme rate corrections (log-linear rule; Hautus, 1995)
- Unequal-variance models (recognition memory zROC slope ≈ 0.80)

**Critical insight**: Percent correct confounds sensitivity and bias. Two observers with identical d' can have different accuracy if their criteria differ (Green & Swets, 1966).

### Reading Time Analysis (references/reading-time-analysis.md)

**When to use**: Analyzing eye-tracking data from sentence or passage reading experiments.

**Key topics**:
- Eye-tracking measure hierarchy (first fixation, gaze duration, go-past time, total time)
- Which measure for which process (early lexical → gaze duration; syntactic reanalysis → go-past time)
- Spillover effects (effects appear 1-2 words downstream)
- Data cleaning (fixation duration cutoffs: 80-800 ms; Rayner & Pollatsek, 1989)
- Statistical modeling (crossed random effects for subjects and items; log-transform or Gamma GLMM)

**Critical insight**: Different measures tap different processing stages. Using only total reading time obscures when effects arose (Clifton et al., 2007).

### Visualization (references/cogsci-visualization.md)

**When to use**: Creating publication-quality figures; choosing plot types and colors.

**Key topics**:
- Plot type selection (raincloud plots for RT, not bar charts; Weissgerber et al., 2015)
- ERP visualization conventions (negative up vs. down; topographic maps)
- fMRI visualization standards (never use jet colormap; Borland & Taylor, 2007)
- Color accessibility (colorblind-safe palettes; viridis, Okabe-Ito)
- Publication formatting (300 DPI, font sizes, within-subject error bars)

**Critical insight**: Bar charts with error bars conceal distribution shape and can obscure effects. Use raincloud plots or violin plots (Allen et al., 2019).

## Common Pitfalls Across Methods

1. **Using generic effect sizes for power analysis**: Cohen's 0.2/0.5/0.8 benchmarks do not reflect cognitive science reality. Use paradigm-specific estimates.

2. **Ignoring items as random effects**: If stimuli are sampled (words, faces, scenes), failing to model item variability inflates Type I error (Clark, 1973).

3. **Conflating sensitivity and bias**: Raw accuracy mixes discrimination ability and response criterion. Use signal detection theory when appropriate.

4. **Using bar charts for continuous data**: Bar charts hide distribution shape. Use raincloud or violin plots (Weissgerber et al., 2015).

5. **Applying behavioral power formulas to neuroimaging**: Whole-brain analyses require simulation-based power that accounts for multiple comparisons (Mumford & Nichols, 2008).

6. **Reporting only total reading time**: Different eye-tracking measures reflect different processing stages. Report first-pass and late measures separately.

7. **Using jet/rainbow colormaps for brain images**: Perceptually non-uniform colormaps create false boundaries. Use viridis or scientifically designed colormaps (Crameri et al., 2020).

## Minimum Reporting Standards

Regardless of method, all analyses should report:

- [ ] Exact test statistics with degrees of freedom
- [ ] Exact p-values (to 3 decimal places or "< .001")
- [ ] Effect sizes with confidence intervals
- [ ] Sample sizes per group/condition
- [ ] Data exclusion criteria and proportion excluded
- [ ] For mixed models: random effects structure, convergence confirmation
- [ ] For multiple comparisons: correction method and justification
- [ ] For power analysis: effect size source, target power, sensitivity analysis
