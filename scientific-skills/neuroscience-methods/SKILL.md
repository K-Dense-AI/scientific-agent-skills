# Neuroscience Methods

## Purpose

This skill provides a unified entry point for seven major neuroscience methodologies spanning systems, experimental, and clinical neuroscience. Each method addresses fundamentally different research questions and requires specialized domain knowledge that general-purpose programmers or data scientists would systematically get wrong without neuroscience training.

The consolidated methods are:

1. **Brain Connectivity Modeling** — Functional and effective connectivity analysis from fMRI/EEG
2. **Calcium Imaging Analysis** — Single-neuron activity extraction from fluorescent indicators
3. **Lesion-Symptom Mapping** — Localizing cognitive functions via patient lesion data
4. **Neural Decoding Analysis** — Multivariate pattern analysis and representational similarity
5. **Neural Population Analysis** — Dimensionality reduction and latent dynamics
6. **Optogenetics Protocol Design** — Causal circuit manipulation via light-activated opsins
7. **Neuropsychological Battery Selection** — Clinical cognitive assessment design

This router skill helps you identify which method matches your research question and directs you to the appropriate detailed guidance.

## Method Overviews

### Brain Connectivity Modeling

**Core distinction**: Functional connectivity (statistical association) vs. effective connectivity (directed causal influence). A competent programmer without neuroscience training would confuse correlation with causation in brain networks and would not appreciate why motion artifacts are devastating for connectivity analyses.

**Key methods**:
- Seed-based correlation and partial correlation for exploratory functional connectivity
- Psychophysiological interactions (PPI) for context-dependent coupling changes
- Dynamic causal modeling (DCM) for hypothesis-driven causal inference (requires strong priors, 3-8 regions maximum)
- Graph theory for network topology (hubs, modules, small-worldness)

**Critical parameters**:
- Motion threshold: FD < 0.2 mm for scrubbing (Power et al., 2014)
- Fisher z-transformation required before group statistics
- Permutation-based FWE correction for multiple comparisons (1000+ permutations)
- Normalize graph metrics against random networks (1000+ null networks)

**Common pitfalls**: Confusing functional connectivity with direct anatomical connection; not controlling for motion; using Granger causality with fMRI (hemodynamic variability confounds causal direction); arbitrary graph thresholds without sensitivity analysis.

**See**: `references/brain-connectivity-modeler.md` for complete decision trees, parameter tables, and DCM model specification.

### Calcium Imaging Analysis

**Core challenge**: Extracting single-neuron signals from fluorescent movies requires handling motion artifacts, neuropil contamination, photobleaching, and indicator kinetics. Standard computer vision approaches fail because they ignore the biophysics of calcium indicators.

**Key pipeline steps**:
1. Motion correction (non-rigid for awake animals)
2. ROI detection (CNMF for 2P, CNMF-E for 1P/miniscope)
3. Neuropil correction (subtract r * F_neuropil, where r ≈ 0.7)
4. dF/F computation (rolling percentile baseline, 30-60 sec window)
5. Deconvolution (OASIS or CASCADE with indicator-matched tau)

**Critical parameters**:
- Neuropil coefficient: r = 0.7 for 2P cortical imaging (Chen et al., 2013)
- Deconvolution tau: GCaMP6s = 1.0-1.5 s, GCaMP6f = 0.4 s, jGCaMP8f = 0.2 s
- SNR threshold: > 3 for reliable transient detection
- Quality metrics: skewness > 0.5, compact spatial footprint

**Common pitfalls**: Wrong tau for your indicator; skipping neuropil correction (inflates correlations); using standard CNMF for 1P data (requires CNMF-E); ignoring photobleaching; interpreting deconvolved amplitudes as absolute spike counts.

**See**: `references/calcium-imaging-analysis-guide.md` for modality-specific pipelines, indicator kinetics tables, and quality control metrics.

### Lesion-Symptom Mapping

**Core challenge**: Lesion distributions are not random (MCA territory bias), lesion volume confounds all analyses, and white matter disconnection matters as much as grey matter damage. Standard voxelwise methods confound truly causal regions with regions that are simply co-damaged.

**Key methods**:
- Voxel-based lesion-symptom mapping (VLSM) for mass-univariate localization (requires N ≥ 50)
- Support vector regression LSM (SVR-LSM) for multivariate patterns (requires N ≥ 100)
- Disconnection analysis (BCBToolkit) for white matter pathway disruption
- Lesion network mapping for functional network disruption

**Critical requirements**:
- Sample size: minimum N = 50, recommended N ≥ 100 (Sperber, 2020)
- Lesion volume control: MUST include as covariate or residualize behavior
- Multiple comparison correction: permutation-based FWE (5000+ permutations)
- Cost function masking during registration (prevents warping artifacts)
- Chronic-phase data preferred (> 3 months post-onset)

**Common pitfalls**: Not controlling for lesion volume (most common error); sample too small (N < 50 produces unreliable maps); not using cost function masking; using uncorrected or Bonferroni thresholds; ignoring vascular territory collinearity.

**See**: `references/lesion-symptom-mapping-guide.md` for statistical test selection, disconnection analysis workflows, and reporting checklists.

### Neural Decoding Analysis

**Core distinction**: Decoding tests whether spatial patterns of activity carry information, even when mean activity is identical across conditions. High decoding accuracy means information is accessible from that region's patterns, NOT that the region is the source of the representation.

**Key methods**:
- Cross-validated classification (MVPA) for presence/absence of information
- Representational similarity analysis (RSA) for comparing neural geometry to models
- Temporal generalization (time × time decoding) for tracking representational dynamics
- Encoding models for predicting neural responses from stimulus features

**Critical parameters**:
- Classifier: linear SVM or logistic regression (nonlinear rarely helps, risks overfitting)
- Cross-validation: leave-one-run-out for fMRI, stratified k-fold for EEG/MEG
- Distance metric for RSA: crossnobis (unbiased) preferred over correlation distance
- Permutation testing: 1000+ label shuffles for significance
- Minimum trials: 20-30 per class for reliable within-subject decoding

**Common pitfalls**: Information leakage (preprocessing outside CV folds); confounds driving decoding (eye movements, RT differences); interpreting accuracy magnitude as neural information amount; double-dipping in searchlight; confusing classifier weights with activation patterns.

**See**: `references/neural-decoding-analysis.md` for RSA workflows, temporal generalization interpretation, and leakage prevention.

### Neural Population Analysis

**Core distinction**: Population analysis characterizes collective low-dimensional structure, not individual tuning curves. PCA finds maximum variance axes; dPCA demixes variance by task parameter; GPFA extracts smooth single-trial trajectories. Choosing the wrong method answers the wrong question.

**Key methods**:
- PCA for dominant co-variation patterns (start here)
- GPFA for smooth latent trajectories on single trials (requires 50-100 neurons)
- dPCA for demixing stimulus, decision, time, and interaction variance
- jPCA for rotational dynamics in neural trajectories

**Critical preprocessing**:
- Soft normalization: (rate - mean) / (range + 5 spikes/s) — NOT standard z-scoring
- Square-root transform alternative: sqrt(rate + constant) for Poisson-like counts
- Mean subtraction: remove condition-averaged PSTH before PCA

**Dimensionality determination**:
- Parallel analysis (compare to shuffled data eigenvalues) — recommended
- Cross-validated PCA (hold out neurons, test reconstruction)
- NOT percent variance threshold (arbitrary and meaningless for neural data)

**Common pitfalls**: Using standard z-scoring (destroys rate information); interpreting t-SNE/UMAP distances as meaningful; using percent variance to choose dimensionality; applying GPFA to trial-averaged data; insufficient neuron counts (< 30 unreliable).

**See**: `references/neural-population-analysis-guide.md` for method comparisons, data requirements, and regularization procedures.

### Optogenetics Protocol Design

**Core challenge**: Selecting an opsin requires understanding photocycle kinetics, ion selectivity, spectral overlap, expression toxicity, and light propagation biophysics. Naive approaches risk tissue heating, depolarization block (silencing when you intend to activate), or uninterpretable results from inadequate controls.

**Key decisions**:
1. Manipulation type: fast excitation, tonic excitation, sustained inhibition, phasic inhibition, bidirectional
2. Opsin selection: ChR2 (standard), ChETA (high-freq), ChrimsonR (red-shifted), stGtACR2 (potent inhibition)
3. Light parameters: 1-10 mW/mm² typical, match wavelength to opsin peak, limit heating to < 1°C
4. Pulse protocol: 1-10 ms pulses, frequency below opsin limit (ChR2 ≤ 40 Hz), duty cycle < 40%
5. Control conditions: opsin-negative + light (non-negotiable), plus at least two others

**Critical parameters**:
- ChR2 EPD50: 1.3 mW/mm² (Mattis et al., 2012)
- Light attenuation: 90% lost within 1 mm at 473 nm
- Heating threshold: keep temperature rise < 1°C (Stujenske et al., 2015)
- Viral titer: 1e12 to 5e12 vg/mL (avoid toxicity)
- Expression time: 3-6 weeks optimal

**Common pitfalls**: Depolarization block at high expression/frequency; tissue heating artifacts; viral toxicity; backpropagation along fibers; chloride loading with halorhodopsin; inadequate controls (most common critique).

**See**: `references/optogenetics-protocol-designer.md` for opsin catalog, pulse protocol recipes, and control condition design.

### Neuropsychological Battery Selection

**Core challenge**: Test selection is clinical judgment, not a checklist. Not all "memory tests" test the same construct. CVLT-II assesses list learning; Logical Memory tests narrative recall; BVMT-R tests visual-spatial memory. Each is sensitive to different lesion profiles.

**Key decisions**:
1. Clarify referral question (dementia differential, TBI, pre-surgical, forensic)
2. Identify target domains (attention, memory, executive, language, visuospatial, motor)
3. Assemble battery (core 2-3 hours, extended 4-6 hours)
4. Select appropriate norms (age, education, premorbid IQ, cultural background)
5. Include validity testing (TOMM, embedded PVTs)

**Core battery components**:
- Premorbid estimate: TOPF or WTAR (~10 min)
- Attention/speed: TMT-A, WAIS-IV Coding, Digit Span (~20 min)
- Executive: TMT-B, verbal fluency, Stroop (~15 min)
- Verbal memory: CVLT-II/III or RAVLT (~30 min)
- Visual memory: BVMT-R or RCFT recall (~25 min)
- Language: BNT (~15 min)
- Visuospatial: RCFT copy or Block Design (~10 min)
- Motor: Grooved Pegboard (~10 min)
- Validity: TOMM Trial 1 or embedded measures (~10 min)

**Common pitfalls**: Redundant tests (CVLT + RAVLT wastes time); wrong norms (education/IQ mismatch); inadequate validity testing; ignoring practice effects in serial assessment; ceiling/floor effects; poor ecological validity.

**See**: `references/neuropsych-battery-selector.md` for condition-specific batteries, normative data selection, and validity test protocols.

## Cross-Method Integration

### Complementary Method Combinations

**Connectivity + Decoding**: Use connectivity to identify networks, then decode to test what information flows through those connections. Example: PPI identifies task-modulated coupling between regions; MVPA tests whether stimulus identity is represented in the coupling pattern.

**Lesion Mapping + Connectivity**: Lesion mapping localizes necessary regions; network lesion mapping identifies the functional network disrupted. Example: VLSM identifies critical voxels for language; lesion network mapping reveals the distributed language network affected.

**Calcium Imaging + Population Analysis**: Calcium imaging provides single-neuron resolution; population analysis reveals collective dynamics. Example: Extract calcium signals from 200 neurons, then apply dPCA to demix stimulus and decision variance.

**Optogenetics + Decoding**: Optogenetics causally manipulates activity; decoding tests representational consequences. Example: Activate PV interneurons, then decode whether stimulus representations are disrupted.

**Neuropsych Battery + Lesion Mapping**: Neuropsych tests quantify deficits; lesion mapping localizes the neural substrate. Example: CVLT-II reveals encoding deficit; VLSM identifies hippocampal damage as the cause.

### Sequential Study Design

**Phase 1 — Localization**: Use connectivity analysis or decoding to identify candidate regions carrying task-relevant information.

**Phase 2 — Necessity**: Use lesion-symptom mapping (in patients) or optogenetics (in animals) to test whether those regions are causally necessary.

**Phase 3 — Mechanism**: Use calcium imaging or population analysis to characterize the neural code and dynamics within the critical regions.

**Phase 4 — Clinical Translation**: Use neuropsychological batteries to assess whether the mechanism generalizes to human cognitive function.

## Minimum Reporting Standards Across Methods

- [ ] Clear statement of research question and hypothesis
